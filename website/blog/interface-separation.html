One Agent, Five Interfaces: Why Separation Matters

CLI, TUI, JSON, RPC, and SDK — five ways to interact with the same agent core. How interface separation keeps q flexible without feature bloat.

q can be used from the command line, in a rich terminal UI, as a JSON-streaming service, through an RPC interface, or embedded as an SDK. Five interfaces, one agent core. Here's why that separation matters and how it works.

The Five Interfaces

CLI

The simplest interface. Text in, text out. Perfect for scripts, CI pipelines, and quick one-shot prompts:

q prompt "Explain this function"
echo "What does this code do?" | q chat
q goal "Fix the timeout issue"

TUI

The terminal user interface provides a rich, interactive experience with syntax highlighting, streaming display, and goal mode visualization:

q tui

Features include file diff previews, session history browsing, and real-time state machine visualization during goal execution.

JSON Mode

For machines. Structured JSON output with the response, tool calls, and events:

echo '{"prompt": "List all TODOs"}' | q json

Ideal for IDE extensions, CI pipelines, and other agents that consume q's output.

RPC

For long-running services. q can be started as an RPC server that other processes connect to:

q serve --port 9876
# Then from another process:
curl http://localhost:9876/ask -d '{"prompt": "..."}'

SDK

Embed q in your own Racket applications:

#lang racket
(require q/sdk)
(define agent (make-agent))
(define response (agent-ask agent "What does this project do?"))

How Separation Works

All five interfaces are pure event consumers. The core agent loop emits events to the bus, and each interface subscribes to the events it cares about:

  • CLI subscribes to text output events
  • TUI subscribes to all events for rich display
  • JSON subscribes to structured events
  • RPC subscribes to response events
  • SDK provides callbacks for specific event types

The Contract

The event contract between core and interfaces is stable and documented:

  • Event types never change their shape (only additive changes)
  • All events include a timestamp and correlation ID
  • Interfaces must handle unknown event types gracefully

This stability means new interfaces can be added without modifying the core — just subscribe to events.

Adding a New Interface

Want to build a web UI for q? You only need to:

  1. Subscribe to the events you want to display
  2. Render them as HTML/WebSocket messages
  3. Send user input back through the agent API

No core modifications needed. That's the power of separation.
