Roadmap to v1.0: What's Next for q

From v0.82 to v1.0: the roadmap for q's journey to a stable release, including GUI, multi-agent collaboration, and the Typed Racket migration.

q v0.82 introduced the goal-directed autonomous loop — our biggest feature yet. But the journey to v1.0 is just beginning. Here's a transparent look at what's coming next, what we're unsure about, and how you can help shape the future of q.

v0.85 — Stability and Polish (Q2 2026)

The next release focuses on making existing features rock-solid:

  • Typed Racket migration continues — more modules get type annotations (see our migration strategy post)
  • Error recovery improvements — better handling of provider failures, network issues, and malformed responses
  • Session branching — fork a session at any point and explore different paths
  • Configuration validation — catch misconfigurations before they cause runtime errors
  • Performance profiling — built-in metrics for identifying bottlenecks in agent loops

v0.90 — GUI (Q3 2026)

The most requested feature: a graphical interface. We're building a native GUI using Racket's built-in GUI toolkit (ADR-17):

  • Code viewer — syntax-highlighted file browser with diff visualization
  • Goal progress dashboard — visual state machine, turn timeline, evidence gallery
  • Extension marketplace — browse and install community extensions
  • Settings GUI — visual configuration instead of editing JSON

The GUI is an extension of the interface separation principle (ADR-5) — it's a new interface that connects to the same agent core. The TUI and CLI aren't going anywhere.

v0.95 — Multi-Agent Collaboration (Q4 2026)

Multiple q instances working together on complex tasks:

  • Task delegation — a coordinator agent breaks work into sub-tasks and delegates to specialist agents
  • Shared context — agents share a common understanding of the codebase
  • Conflict resolution — when two agents modify the same file, a merge strategy resolves conflicts
  • Role specialization — "reviewer", "implementer", "tester" agents with different capabilities

This is the most uncertain part of the roadmap. Multi-agent coordination is a hard problem, and we may revise the approach based on research and prototyping.

v1.0 — API Stability (Q1 2027)

v1.0 means API stability guarantees:

  • Stable extension API — extensions written for v1.0 will work in v1.x without modification
  • Stable event schema — event types and their fields won't change incompatibly
  • Stable configuration format — existing config.json files will continue to work
  • Comprehensive documentation — every public API documented with examples
  • Migration guides — clear upgrade path from any 0.x version

What's Not on the Roadmap

Equally important — things we're not planning:

  • Cloud hosting — q will remain local-first. Cloud features may come as opt-in additions
  • Proprietary extensions — all core features remain open source
  • Electron UI — we're committed to native rendering
  • Model training — q is a tool for using models, not training them

How You Can Help

The roadmap isn't set in stone. We prioritize based on community feedback:

  • File issues — bugs, feature requests, documentation gaps
  • Submit PRs — especially for the Typed Racket migration and test coverage
  • Write extensions — test the extension API and tell us what's missing
  • Share use cases — tell us how you use q and what would make it better

Release Cadence

We aim for:

  • Patch releases (0.82.x) — bug fixes, weekly or as needed
  • Minor releases (0.83, 0.84, ...) — new features, monthly
  • Major milestone releases (0.85, 0.90, 0.95) — significant new capabilities, quarterly
"A roadmap is a plan, not a promise. We adapt based on what we learn and what the community needs. Transparency about our direction matters more than hitting every date." — q contributors
