🧩 Extension Guide

Build custom tools, hooks, and integrations using q's extension system.

Hook System Overview

q exposes a hook system with named dispatch points (ADR-6). Extensions register handler functions for specific hooks. Each handler returns one of:

  • pass — continue without modification
  • block — abort the action with an optional error message
  • modify — transform the data and continue

Hooks execute in registration order. The core invokes hooks at clearly documented points in the agent loop.

Available Hooks

Hook NameWhen FiredCan Modify
session-startWhen a new session beginsSession config
pre-tool-callBefore a tool is executedTool arguments
post-tool-callAfter a tool returnsTool result
pre-promptBefore sending to the LLMPrompt content
post-responseAfter receiving LLM responseResponse content
context-assemblyWhen building context for a turnContext structure

Creating an Extension

Extensions live in extensions/<name>/ and follow a standard structure:

extensions/
  my-extension/
    manifest.json       # Extension metadata
    main.rkt            # Entry point - registers hooks
    hooks/
      pre-tool.rkt      # Hook handlers

Extension Manifest

{
  "name": "my-extension",
  "version": "1.0.0",
  "description": "Custom extension for my workflow",
  "hooks": ["pre-tool-call", "post-response"],
  "dependencies": []
}

Hook Handler Example

#lang racket

(provide register-hooks)

(define (register-hooks hook-dispatcher)
  (hook-dispatcher 'register
    "pre-tool-call"
    (λ (tool-name args)
      (cond
        [(equal? tool-name "shell")
         (define cmd (hash-ref args 'command ""))
         (when (dangerous-command? cmd)
           (values 'block (format "Blocked dangerous command: ~a" cmd)))]
        [else (values 'pass #f)]))))

(define (dangerous-command? cmd)
  (regexp-match? #rx"(rm -rf /|format |del /)" cmd))

Built-in Extensions

q ships with several built-in extensions in extensions/:

  • gsd/ — Goal-directed planning system (GSD state machine)
  • github/ — GitHub integration (PRs, issues, reviews)
  • git/ — Git operations with safety checks
  • hooks/ — Core hook infrastructure
💡 Extension isolation

Extensions run in the same Racket process but are isolated through the hook system. A crashing extension cannot bring down the agent — errors are caught and logged.

SDK & RPC Integration

For deeper integration, use the SDK (interfaces/sdk-core.rkt):

#lang racket

(require q/interfaces/sdk-core)

;; Create a programmatic session
(define session (make-session
  #:provider "openai"
  #:model "gpt-5.4"))

;; Send a prompt and get streaming results
(define result (session-send session "analyze this code"))

;; Subscribe to events
(session-on-event session
  (λ (event)
    (printf "Event: ~a~n" (event-type event))))
