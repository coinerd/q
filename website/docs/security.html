🔐 Security & Trust Model

Security-first design with defense-in-depth: sandboxing, credential redaction, append-only logs, and clear trust boundaries.

Security Overview

q executes LLM-generated code and shell commands on your machine. This creates inherent risk. q's security architecture (spanning ADR-1, ADR-2, ADR-7, ADR-8, ADR-9) addresses this through defense-in-depth: multiple independent layers that each reduce the attack surface.

Trust Boundaries

q defines explicit trust boundaries:

  • Core ↔ Tools — tool execution is sandboxed; tools cannot access session state directly
  • Core ↔ Extensions — extensions run in a constrained hook system; they can pass/block/modify, not execute arbitrary code
  • Core ↔ Interfaces — interfaces are pure event consumers; they receive events but cannot influence the agent loop
  • Core ↔ Providers — API keys are resolved at the boundary and never passed to tool execution

Command Sandboxing

Shell commands generated by the LLM are executed in a sandboxed subprocess (ADR-7):

  • Shell quoting — all arguments are properly quoted to prevent injection
  • Resource limits — configurable memory and CPU limits
  • Timeouts — commands are killed after a configurable timeout (default: 30s)
  • Output truncation — output is capped at max-output-bytes to prevent memory exhaustion
  • Working directory scoping — commands run within the project directory
# Tool configuration in config.json
"tools": {
  "shell-timeout": 30,
  "max-output-bytes": 65536,
  "file-scope": "."
}

Credential Redaction

API keys are never stored in plaintext in session logs (ADR-9):

  • Resolution chain — env var → OS keychain → file (in order of preference)
  • Log redaction — keys are replaced with [REDACTED] before writing to JSONL
  • Event scrubbing — all events emitted to the bus have keys stripped
  • File permissions — credential files must have restricted permissions (chmod 600)

Append-Only Session Logs

Every action is recorded in JSONL format (ADR-2):

  • The agent loop only appends — never rewrites or deletes log entries
  • Each event includes a timestamp, session ID, and event type
  • Logs are tamper-detectable via sequential event numbering
  • Logs provide a complete audit trail for forensic analysis

Safe Mode

q supports a safe mode (ADR-8) that restricts available tools:

  • Disables shell command execution
  • Disables file write operations
  • Limits to read-only tools (file read, search, code analysis)
  • Useful for code review and analysis workflows

Extension Security

The extension hook system (ADR-6) provides security boundaries:

  • Extensions register for named hooks — they cannot intercept arbitrary code paths
  • Hook handlers run in registration order with clear return semantics
  • A crashing extension doesn't bring down the agent (defensive error handling)
  • Extensions can block actions (e.g., prevent a dangerous command from executing)
