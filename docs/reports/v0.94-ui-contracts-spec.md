# v0.94 UI Actions and Component Contracts

**Version:** 0.94.0-draft
**Date:** 2026-06-05
**Status:** W0.3 contract freeze — defines the target contracts for v0.94.1–v0.94.7

## 1. UI Action Event Names

All UI action events follow the `ui.*` namespace. They are published to the event bus as regular events with string type names.

### 1.1 Core UI Actions

| Event Name | Payload Keys | Source | Consumer |
|---|---|---|---|
| `ui.header.set` | `lines: (listof any/c)` | extension/ui-surface | TUI adapter, GUI adapter |
| `ui.header.clear` | (none) | extension/ui-surface | adapters |
| `ui.footer.set` | `lines: (listof any/c)` | extension/ui-surface | adapters |
| `ui.footer.clear` | (none) | extension/ui-surface | adapters |
| `ui.status.message` | `message: string?` | dialog-api | adapters |
| `ui.theme.change` | `theme: ui-theme?` | theme-manager / CLI | adapters |
| `ui.layout.breakpoint` | `breakpoint: (or/c 'narrow 'standard 'wide 'tall)` | layout-policy | adapters |
| `ui.focus.change` | `component: symbol?` | dispatch | adapters |
| `ui.scroll` | `direction: (or/c 'up 'down 'top 'bottom)` | dispatch | adapters |
| `ui.resize` | `cols: exact-positive-integer?`, `rows: exact-positive-integer?` | dispatch | adapters |
| `ui.widget.register` | `ext-name: symbol?`, `key: any/c`, `descriptor: hash?` | widget-api | adapters |
| `ui.widget.unregister` | `ext-name: symbol?`, `key: any/c` | widget-api | adapters |
| `ui.widget.unregister-all` | `ext-name: symbol?` | widget-api | adapters |

### 1.2 User Input Actions (already defined in ui-core/dispatch.rkt)

| Event Name | Payload Keys |
|---|---|
| `user.input` | `text: string?` |
| `user.cancel` | (none) |
| `user.command` | `command: string?`, `args: (listof string?)` |

## 2. UI Delta Shape

The shared UI reducer produces a `ui-delta` struct:

```racket
(struct ui-delta (type payload) #:transparent)
;; type : symbol? — one of 'set-header, 'clear-header, 'set-footer, 'clear-footer,
;;                        'set-status, 'set-theme, 'add-message, 'update-message,
;;                        'set-focus, 'set-layout, 'register-widget, 'unregister-widget
;; payload : any/c — type-specific data
```

Backend adapters consume `ui-delta` values and apply them to backend-specific state.

### 2.1 Delta Payload Shapes

| Delta Type | Payload Shape |
|---|---|
| `set-header` | `(listof styled-line)` |
| `clear-header` | `#f` |
| `set-footer` | `(listof styled-line)` |
| `clear-footer` | `#f` |
| `set-status` | `symbol?` (`idle`, `processing`, `error`) |
| `set-theme` | `ui-theme?` |
| `add-message` | `gui-message?` (reused for both TUI and GUI) |
| `update-message` | `(cons index gui-message?)` |
| `set-focus` | `symbol?` |
| `set-layout` | `gui-layout?` |
| `register-widget` | `(list ext-name key descriptor)` |
| `unregister-widget` | `(cons ext-name key)` |

## 3. Backend Adapter Contract

```racket
;; adapter : ui-delta -> backend-state -> backend-state
;; Pure function: takes delta and current backend state, returns new state.
;; No side effects. No IO.
```

Each backend (TUI, GUI) provides an adapter function matching this signature.

### 3.1 TUI Adapter

- Consumes `ui-delta` values
- Transforms `ui-state` (from `tui/state-types.rkt`)
- Must NOT import `gui/` or `racket/gui`

### 3.2 GUI Adapter

- Consumes `ui-delta` values
- Transforms `gui-state` (from `gui/gui-types.rkt`)
- Must NOT import `tui/`

## 4. Extension Widget Descriptor Schema

```racket
;; Widget descriptor (minimal v1):
(hash 'type (or/c 'text 'lines 'progress 'status)
      'content any/c        ; type-specific content
      'priority exact-integer? ; ordering hint (higher = more prominent)
      'trust-level (or/c 'trusted 'untrusted) ; safe-mode gate
      'zone (or/c 'sidebar 'header 'footer 'overlay 'transcript))
```

All fields are optional except `type` and `content`. Adapters must validate before rendering.

## 5. Hook Timeout and Error Policy

### 5.1 GUI Lifecycle Hooks

| Hook | Timeout | Error Policy |
|---|---|---|
| `gui.window.opened` | 100ms | Log and continue |
| `gui.window.closed` | 100ms | Log and continue |
| `gui.theme.changed` | 50ms | Log, skip theme update |
| `gui.layout.changed` | 50ms | Log, skip layout update |
| `gui.focus.changed` | 50ms | Log and continue |
| `gui.input.submit` | 200ms | Log, proceed with original input |
| `gui.command.execute` | 200ms | Log, proceed with original command |
| `gui.scroll.request` | 50ms | Log and continue |

### 5.2 Render Hooks (v0.94.7 target)

| Hook | Timeout | Error Policy |
|---|---|---|
| `ui.component.pre-render` | 50ms | Skip hook, render default |
| `ui.component.post-render` | 50ms | Log and continue |

## 6. Compatibility/Fallback Rules

During v0.94.xx, legacy callback APIs remain supported:

1. **If event-driven UI enabled and event bus available:** publish `ui.*` action event
2. **Else:** use legacy callback path (current behavior)

The feature flag is `current-ui-event-driven?` (parameter, default `#f`).

Transition plan:
- v0.94.1: Introduce flag and dual-path in `ui-surface.rkt`
- v0.94.2: Prove adapters produce identical observable effects via parity tests
- v0.94.3+: Flip default to `#t` after parity is proven
- Post-v0.94: Remove legacy callback path

## 7. Module Dependency Rules

### 7.1 `ui-core/` MUST only import from:
- `ui-core/` (sibling modules)
- `util/` (shared utilities)
- `racket/base`, `racket/contract`, `racket/hash`, etc.

### 7.2 `ui-core/` MUST NOT import from:
- `tui/`
- `gui/`
- `extensions/`
- `runtime/`
- `agent/`
- `tools/`

### 7.3 Adapters MAY import from:
- `ui-core/`
- Their own backend module (`tui/` for TUI adapter, `gui/` for GUI adapter)
- `util/`
