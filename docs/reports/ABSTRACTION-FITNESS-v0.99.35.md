## Purpose

This report provides repeatable, evidence-based abstraction fitness signals for
the q-agent codebase. It is generated by:

```bash
racket scripts/abstraction-audit.rkt --root . --out <report> --json-out <json>
```

The tool supplements the hand-curated abstraction inventory
(`.planning/ABSTRACTION-INVENTORY-v0.99.35.md`). It is advisory by default and
only fails the build when invoked with `--strict`.

## Signal Categories

1. **Line count** — proxy for module complexity
2. **Export count** — proxy for public API surface area
3. **Parameter usage** (`make-parameter`/`parameterize`) — dynamic context usage
4. **Macro usage** (`define-syntax`/`syntax-parse`/`syntax-case`/`define-syntax-rule`)
5. **struct-out exports** — public struct internals exposed
6. **I/O mixed with pure logic** — files that have both `(define ...)` and file/process I/O
7. **Serialization hotspots** — `->jsexpr`/`jsexpr->`/`->hash`/`hash->` patterns
8. **Exception handler density** — `with-handlers` count
9. **Dependency fan-out** — `(require ...)` form count
